# Game studio subagents

Fourteen specialist roles that collaborate to build Swift iOS/iPadOS games with the
`swift-ios-game-studio` skill. **This folder is the canonical source of truth.** Tool-specific
copies are generated by `sync-agents.py`:
- **Claude Code:** `.claude/agents/<name>.md` (native subagents — invoke by name or let Claude route).
- **Cursor:** `.cursor/rules/agents/<name>.mdc` (per-role rules / custom-mode prompts).
- **Codex:** these files + the "Subagents" section in the repo-root `AGENTS.md`.

## The team (14 specialists)
**Build roles**
| Agent | Role | Typically called |
|---|---|---|
| `game-coordinator` | PM: decompose the task, sequence work, decide who's next | **first** |
| `game-designer` | mechanics, core loop, progression, economy concept, Mini-GDD | after coordinator |
| `engine-architect` | mode (SwiftUI/SpriteKit/hybrid), architecture, perf budget, seams | after designer |
| `gameplay-programmer` | implement systems, abilities, interaction/combat, UI flow | after architect |
| `art-director` | original art: vector/procedural 2D, sprites/atlases, light 3D, palettes | parallel w/ programmer |
| `narrative-writer` | quests, copy, lore, tutorial flow, localized strings | parallel w/ programmer |
| `balance-economist` | difficulty/progression curves, economy, win-rate/tempo modeling | parallel w/ programmer |
| `qa-tester` | test cases, unit tests, edge cases, accessibility, runs build/tests | after programmer |

**Review, audit & release roles**
| Agent | Role | Typically called |
|---|---|---|
| `code-reviewer` | reviews a single **diff/PR**: bugs, style, architecture vs the skill + Swift bar | per change |
| `code-auditor` | sweeps the **whole codebase**: systemic correctness, consistency, dead code, coverage | milestones / pre-release |
| `security-auditor` | data leaks, insecure storage/network, secrets, permissions, **kids-privacy** | before release |
| `performance-auditor` | frame budget, allocations, draw calls, memory, battery; profiling plan | before release |
| `legal-compliance` | App Store guidelines, COPPA/GDPR-K, Kids Category, asset/dep **licensing**, IP | before release |
| `release-engineer` | submission-readiness: icons, Info.plist, archive/export, App Store Connect, TestFlight | at release |

## Pipeline
```
coordinator → designer → architect → ┬─ programmer ──┐→ qa-tester → code-reviewer ──┐
                                      ├─ art-director │   (loop back on failures)    │
                                      ├─ narrative    │                              ▼
                                      └─ balance      ┘   pre-release audit gate:
                                                          code-auditor · security-auditor ·
                                                          performance-auditor · legal-compliance
                                                                       │
                                                                       ▼
                                                          release-engineer → submit
```
The audit roles are read-only — they report findings and route fixes back to the build roles;
`release-engineer` then prepares the submission.

## How to invoke
- **Claude Code:** `@game-designer …` or just describe the need — Claude auto-routes by the
  agent `description`. Subagents can't spawn each other, so the coordinator returns a plan that
  you (or the main thread) execute step by step.
- **Cursor:** open the matching rule in `.cursor/rules/agents/`, or paste it as a Custom Mode
  system prompt (Settings ▸ Chat ▸ Custom Modes).
- **Codex:** ask it to "act as the <role> agent" — it reads the spec here via `AGENTS.md`.

## Editing
Edit the files **here**, then regenerate the tool copies:
```bash
.agents/agents/sync-agents.py            # write .claude/agents + .cursor/rules/agents
.agents/agents/sync-agents.py --check    # CI: fail if any copy drifted
```

## Shared contract (every agent enforces)
No copyrighted assets · testable UI-free logic core · accessibility on all controls · kids
safety & privacy (no tracking/ads/external-links/accounts, offline-first) · the Swift quality bar
in `../skills/swift-ios-game-studio/references/swift/README.md` · **no App Store/compliance
guarantees** (checklists and risks only).
